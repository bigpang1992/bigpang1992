一直想在本地跑一个大模型，不用联网不怕审查不花钱。上周终于动手了，选了 DeepSeek-V3 的量化版，用 Ollama 部署。

记录一下全过程，给同样想折腾的兄弟省点时间。

## 我的配置

- CPU：i7-13700K
- 内存：32GB DDR5
- 显卡：RTX 4070Ti 12GB
- 系统：Windows 11

先说结论——这个配置跑 DeepSeek 的 7B 量化版完全没问题，跑 32B 就很勉强了。

## 安装步骤

### 第一步：装 Ollama

直接去 [ollama.com](https://ollama.com) 下载安装包，一路 Next 就行。装完在终端输入 `ollama --version` 确认一下。

### 第二步：拉模型

```bash
ollama pull deepseek-v3:7b-q4
```

7B 的 Q4 量化版大概 4GB 左右，我的网速下载了十来分钟。

### 第三步：跑起来

```bash
ollama run deepseek-v3:7b-q4
```

到这一步就能聊了。输入中文直接对话，体验跟用网页版差不多。

### 第四步（可选）：装个前端

光在终端里聊天也太极客了，装个好看点的前端界面。我用的 Open WebUI：

```bash
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway ghcr.io/open-webui/open-webui:main
```

打开 `localhost:3000` 就能看到一个类 ChatGPT 的界面了。

## 踩坑记录

### 坑1：显存不够

我一开始贪心下了 32B 的版本，结果一运行显存直接爆了，系统卡了大概十秒才缓过来。12GB 显存老实跑 7B 吧。

### 坑2：中文回答质量

说实话，7B 的中文回答质量跟在线版比差距挺大的。复杂一点的问题会出现答非所问、逻辑不通顺的情况。日常聊天够用，写代码也凑合，但指望它替代 GPT-4 那是想多了。

### 坑3：Ollama 默认只监听本地

如果你想在同一局域网的其他设备访问，需要设置环境变量 `OLLAMA_HOST=0.0.0.0`。这个坑让我折腾了半小时才发现。

## 速度实测

用 7B-Q4 版本，4070Ti 加速：

| 场景 | 首字延迟 | 生成速度 |
|------|----------|----------|
| 简单问答 | ~1秒 | ~40 token/s |
| 写一段代码 | ~2秒 | ~35 token/s |
| 长文生成 | ~2秒 | ~30 token/s |

体感上还行，不算快但能接受。比在线版好的一点是不会排队、不会断线。

## 值得本地部署吗

看你的需求：

**值得的情况：**
- 注重隐私，不想把对话数据传到服务器
- 网络环境不好，经常断网
- 想折腾，享受 DIY 的乐趣
- 有离线使用场景

**不值得的情况：**
- 对回答质量要求高（7B 真的比在线版差挺多的）
- 不想折腾环境（虽然 Ollama 已经很简单了）
- 电脑配置低（8GB 以下显存就别想了）

## 后续玩法

1. 接入到 VSCode 里辅助写代码
2. 做个知识库，用 RAG 让它读你的本地文档
3. 搭个 API，给自己的其他项目用
