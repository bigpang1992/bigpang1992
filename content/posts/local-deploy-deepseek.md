一直想在本地跑一个大模型，不用联网不怕审查不花钱。上周终于动手了，选了 DeepSeek-V3 的量化版，用 Ollama 部署。

记录一下全过程，给同样想折腾的兄弟省点时间。

先声明：这篇文章的内容基于我的个人硬件和网络环境，不同配置可能体验差异很大。但踩过的坑应该是通用的。

## 为什么选DeepSeek

市面上可以本地部署的开源模型挺多的——Llama 3.x系列、Mistral、Qwen……我最后选DeepSeek-V3主要是三个原因：

1. **中文能力强**：DeepSeek在中文理解和生成方面做得很好，中文场景下的表现比同参数量的Llama强不少
2. **国产模型加buff**：说实话有点情怀在——支持国产AI嘛
3. **社区活跃**：Ollama对DeepSeek的支持很完善，网上教程也多

当然也有考虑过Qwen2.5（阿里的），中文也很强。但DeepSeek的量化版在我这个显存配置下表现更稳定。

## 我的配置

- CPU：i7-13700K
- 内存：32GB DDR5
- 显卡：RTX 4070Ti 12GB
- 系统：Windows 11
- 硬盘：M.2 NVMe SSD（模型文件加载速度跟硬盘有关系）

先说结论——这个配置跑 DeepSeek 的 7B 量化版完全没问题，跑 32B 就很勉强了。具体对比后面会说。

**最低配置参考**：如果你的显卡只有8GB显存，跑7B的Q4量化版也是可以的，只是速度会慢一些。如果是集显或者4GB以下的显卡，那就只能CPU推理了，速度会慢很多（大概5-10 token/s），日常聊天勉强能用但等得难受。

## 安装步骤

### 第一步：装 Ollama

直接去 [ollama.com](https://ollama.com) 下载安装包。Windows版就是一个exe，一路 Next 就行。装完在终端输入 `ollama --version` 确认一下。

Ollama本质上是一个大模型的管理工具——它帮你处理模型下载、量化、推理等一系列繁琐的事情，让你不用自己折腾Python环境和各种依赖。可以理解为大模型界的Docker。

**Tips**：装完之后Ollama会在系统托盘里常驻运行。如果你不想让它一直后台跑，可以在托盘图标右键退出。下次要用的时候手动启动就行。

### 第二步：拉模型

```bash
ollama pull deepseek-v3:7b-q4
```

7B 的 Q4 量化版大概 4GB 左右，我的网速下载了十来分钟。

**关于量化等级的科普**：
- **Q4**（4-bit量化）：体积最小、速度最快，但精度有一定损失
- **Q5**：精度和体积的折中，推荐选项
- **Q8**（8-bit量化）：精度接近原版，但体积翻倍，对显存要求更高
- **FP16**（半精度）：基本无损，但7B就需要14GB显存，一般消费显卡跑不起

对于日常使用来说，Q4就够了。如果你显存充裕（16GB以上），可以试试Q5或Q8。

### 第三步：跑起来

```bash
ollama run deepseek-v3:7b-q4
```

到这一步就能聊了。输入中文直接对话，体验跟用网页版差不多（虽然智力上有差距）。

终端界面比较极客，但功能够用：直接打字就是对话，输入`/bye`退出，输入`/help`看更多命令。

### 第四步（可选但强烈推荐）：装个前端

光在终端里聊天也太极客了，装个好看点的前端界面。我用的 Open WebUI：

```bash
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway ghcr.io/open-webui/open-webui:main
```

打开 `localhost:3000` 就能看到一个类 ChatGPT 的界面了。注册一个管理员账号，然后在设置里选择你的Ollama模型就行。

**没装Docker怎么办？** 可以用另一个轻量前端——Chatbox（chatboxai.app），直接下载桌面客户端，在设置里连接Ollama的API地址（默认是 `http://localhost:11434`）。不需要Docker，比Open WebUI好装很多。

**还有一个选择**：如果你用VS Code，可以装Continue插件，直接在编辑器里调用本地模型辅助编程。配置方法是在Continue的config文件里加上Ollama的endpoint。这样你的代码辅助也完全本地化了——零延迟、零费用、零数据泄露。

## 踩坑记录

### 坑1：显存不够导致系统假死

我一开始贪心下了 32B 的版本。`ollama run deepseek-v3:32b-q4` 一运行——屏幕冻住了。

鼠标不动、键盘没反应，整个系统卡了大概十秒才缓过来。Windows弹了一个"GPU资源不足"的通知。

原因很简单：32B的Q4量化版需要大约18GB显存，我的4070Ti只有12GB。Ollama默认会尝试把整个模型加载到显存，装不下就会导致系统级的资源竞争。

**解决方案**：12GB 显存老老实实跑 7B。如果实在想跑32B，可以用CPU+GPU混合推理：

```bash
OLLAMA_NUM_GPU=20 ollama run deepseek-v3:32b-q4
```

`OLLAMA_NUM_GPU=20` 的意思是只把前20层放进GPU，剩下的用CPU跑。缺点是速度大幅下降——从40 token/s降到大概8 token/s。将就能用，但体验不太好。

### 坑2：中文回答质量差距明显

说实话，7B 的中文回答质量跟在线版比差距挺大的。这是参数量决定的物理限制——7B的大脑容量就那么大，塞不下太多知识。

**具体表现**：
- 简单问答（"Python怎么读取CSV文件"）→ 回答质量不错，代码能直接用
- 中等难度（"帮我写一个React的自定义Hook处理分页"）→ 能写出来但有小bug，需要修修改改
- 复杂问题（"分析一下微服务和单体架构各自的适用场景"）→ 答案浅薄，像在背书，缺少有深度的分析
- 中文创意写作 → 比英文差很多，语言不够自然

总结：日常聊天和简单编程任务够用，但指望它替代 ChatGPT Plus / Claude 那是想太多了。

### 坑3：Ollama 默认只监听本地

如果你想在同一局域网的其他设备（比如iPad）访问你电脑上跑的模型，需要设置环境变量：

```
OLLAMA_HOST=0.0.0.0
```

Windows的话，在系统环境变量里添加这一项，然后重启Ollama服务。

这个坑让我折腾了半小时才发现。一开始我以为是防火墙问题，关了Windows防火墙也连不上。最后查了Ollama的GitHub Issues才知道是默认只绑定了localhost。

**安全提醒**：设置 `0.0.0.0` 后你的模型API对局域网完全开放，任何人都能调用。如果你在公司网络或合用Wi-Fi的环境，建议加上API密钥或者只在私有网络使用。

### 坑4：模型切换后旧模型占内存

如果你下载了多个模型（比如7B和一个3B的小模型），在`ollama run`切换模型的时候，旧模型可能还留在内存里没释放。

解决方法：先 `ollama stop 模型名` 再启动新模型，或者直接 `ollama rm` 删掉不用的模型。

## 速度实测

用 7B-Q4 版本，4070Ti GPU 加速：

| 场景 | 首字延迟 | 生成速度 | 回答质量 |
|------|----------|----------|----------|
| 简单问答 | ~1秒 | ~40 token/s | 不错 |
| 写一段代码 | ~2秒 | ~35 token/s | 中等 |
| 长文生成 | ~2秒 | ~30 token/s | 一般 |
| 逻辑推理 | ~2秒 | ~25 token/s | 较差 |

体感上还行，不算快但能接受。比在线版好的一点是：**不会排队、不会断线、没有审查**。

对比一下在线版ChatGPT（GPT-5.2）：
- 首字延迟更短（网络好的话<1秒）
- 回答质量高出一大截
- 但偶尔会排队、会报错、会因为"内容政策"拒绝回答

所以本地模型和在线模型是**互补关系**而不是替代关系。

## 值得本地部署吗

看你的需求：

| 需求 | 推荐 |
|------|------|
| 隐私第一、不想数据上传 | ✅ 非常推荐本地 |
| 需要最强智力水平 | ❌ 买ChatGPT Plus |
| 预算有限、长期大量使用 | ✅ 本地长期免费 |
| 想学习AI原理、折腾玩玩 | ✅ 本地部署是最好的学习方式 |
| 日常办公辅助 | ⚠️ 看具体场景 |
| 编程辅助 | ⚠️ 简单任务够用 |

### 我的实际使用方式

部署完之后用了两周，我的使用习惯慢慢定型了：

- **本地模型处理的**：快速代码片段生成、正则表达式、数据格式转换、随意的闲聊提问、任何我不想让第三方看到的内容
- **在线模型处理的**：复杂编程任务、长文写作、需要联网搜索的信息查询、深度技术分析

这样分工后，我的ChatGPT Plus用量减少了大概30%——那些简单的、不需要太高智力的任务全部交给本地模型了。

## 后续折腾计划

本地部署完之后还有很多可以玩的：

1. **试试用LoRA微调** — 用自己的数据微调一个专属的小模型，比如我打算用我写过的所有代码来训练，让它更适应我的编程风格
2. **搭建RAG系统** — 让本地模型可以读取本地文档（比如项目文档、笔记），然后基于这些文档回答问题
3. **多设备共享** — 在NAS或者一台专门的工作站上跑模型，所有设备通过局域网访问
4. **接入自动化工作流** — 用n8n或者Make把本地模型接入日常工作流，比如自动处理收到的邮件
5. **试试其他模型** — Qwen2.5、Llama 3.3、Mistral等，不同模型在不同任务上表现不同

折腾本地AI最大的乐趣就是——**一切由你掌控**。模型在你的硬盘上，数据在你的内存里，不依赖任何第三方。在这个数据隐私越来越敏感的时代，这种掌控感很珍贵。
