我在 ChatGPT 和 Claude 之间来回切换了至少四次。每次切换都觉得"这次终于找到真爱了"，然后过两周又开始觉得"还是那个好"。

今天来复盘一下这段堪比渣男的经历。如果你也在纠结用哪个，这篇应该能帮你省点时间（和订阅费）。

## 第一阶段：ChatGPT 初恋期（2024年初）

2024 年，跟大部分人一样，ChatGPT 是我的第一个 AI 工具。当时觉得太牛了，什么都能答，什么都能写，感觉整个世界都被打开了。

那时候的使用场景很简单：
- 写邮件时让它帮我润色
- 遇到不会的技术问题直接问
- 无聊的时候跟它扯淡
- 让它帮我解释一些我看不懂的英文文档

那时候的感受：**这就是未来啊！**

那时候也没得选——Claude 还没有广泛开放给普通用户，Gemini 刚起步表现平平。ChatGPT 就是唯一的"AI正宫"。

## 第二阶段：遇到Claude，发现新大陆（2024年中）

后来工作中开始更多地用 AI 辅助编程，发现 ChatGPT 在长代码场景下有个毛病——**越聊越傻**。

具体表现：
- 一开始你跟它定好了约定："用TypeScript写，组件用函数式，不要用class"
- 聊了十几轮之后，它突然给你来了一个class component
- 你提醒它，它说"对不起"，然后继续用class
- 上下文长了之后，前面商量好的代码变量名、函数签名也全忘了
- 最离谱的是有一次它把我项目的框架从React写成了Vue，之前二十几轮对话全白费

这就很崩溃。你花了大量时间给它构建上下文，结果它到后面自己全丢了。每次都要从头重新讲一遍需求，严重影响效率。

朋友推荐我试 Claude，我一试就被惊到了。

### Claude的第一印象

Claude 对长上下文的理解能力明显更强——你在对话开头定义的规则，到了第三十轮它还记得。代码风格一致性也更好，写出来的东西跟你前面的代码在一个风格体系下。

但最让我感动的是——**它会说不知道**。

有一次我问它一个比较冷门的npm包的用法。ChatGPT信誓旦旦地给了一段代码，我抄了之后报错，查了半天发现那个API根本不存在——它编的。同样的问题问Claude，Claude诚实地说："这个包我不太熟悉，建议查看官方文档。如果你能把文档内容贴给我，我可以帮你分析用法。"

就这一个对比，Claude的可靠性在我心里拔高了好几个档次。

另外Claude的文风也更舒服。ChatGPT有时候会很"公事公办"，答案像从教科书里抄的。Claude更像一个认真思考过的同事在跟你讨论——会承认不确定的地方，会提出替代方案，会解释自己的推理过程。

那时候的感受：**之前浪费的时间都是什么！**

## 第三阶段：又回 ChatGPT（2024年底-2025年初）

用了几个月 Claude 之后，发现它也有明显短板。而且这些短板在日常使用中越来越让人难受：

### Claude的短板

**1. 不能联网搜索**

那时候Claude还不能联网（后来加了，但刚加的时候效果不太行）。很多时候我需要查最新的技术信息——"React 19有什么新特性"、"某个包的最新版本怎么用"——Claude只能基于训练数据回答，信息可能是过时的。

**2. 不能生成图片**

ChatGPT有DALL·E，想生成个图直接在对话里就能描述需求生成。Claude啥也没有。偶尔需要做个简单的配图或者概念设计的时候，就只能切回ChatGPT。

**3. 插件/工具生态差距大**

ChatGPT有Code Interpreter（现在叫Advanced Data Analysis），可以直接上传文件、运行Python代码、生成图表。这对于做数据分析的人来说太方便了。Claude虽然也支持文件上传，但没有运行代码的环境。

**4. 有时候太啰嗦**

Claude有一个"好处"也是"坏处"——它太谨慎了。一个简单的问题，它也要告诉你"这只是我的建议，具体情况请自行判断"、"我的训练数据截至到某某日期，以上信息可能不是最新的"。偶尔一次没问题，每次都加这些disclaimer就很烦。

相比之下ChatGPT更干脆——问什么答什么，不啰嗦。虽然有时候过于自信了（上面提到的幻觉问题），但用起来确实更爽快。

### 回去的直接原因

GPT加了Canvas功能，编辑文档的体验相当不错——在对话窗口旁边直接打开一个编辑器，AI可以帮你改写、扩展、调整特定段落，你也可以手动编辑。写长文档的时候这个功能爽到飞起。

Claude的Artifacts也不错，但在交互流畅度上还是差了一些。

那时候的感受：**果然还是 ChatGPT 功能全面！**

## 第四阶段：再回 Claude（2025年末至今）

GPT 用了一阵子，历史毛病又犯了——**幻觉（Hallucination）**。

### 那次让我彻底炸裂的幻觉

有一次我让它帮我查一个 npm 包的用法——`date-fns` 的 `formatRelative` 函数。它信誓旦旦地给了我一段示例代码，说可以传入第三个参数来自定义locale。

我照着抄，怎么都报错。`TypeError: formatRelative expects 2-3 arguments`——等等，明明是支持第三个参数的，为什么报错？

排查了半小时，终于发现——它给的第三个参数的格式是错的。它编了一个不存在的配置对象结构，看起来完全合理，连属性名都取得很像回事。但实际的API根本不是这样传参的。

同样的问题问 Claude，它说："`formatRelative` 确实支持第三个参数，是一个包含 `locale` 属性的选项对象。不过我不确定最新版本是否有API变化，建议查看 [date-fns的官方文档](https://date-fns.org) 确认一下。"

然后它给了一个**基本正确**的示例代码。虽然不是100%完美，但至少方向是对的，稍微一调就能用。

### 不止这一次

还有一次更离谱。我让ChatGPT帮我查一个GitHub仓库的Star数和最新版本号。它不仅给了一个完全错误的Star数（差了两个数量级），还编了一个根本不存在的最新版本号。

ChatGPT有一个问题是——**它的自信程度和正确率之间没有任何相关性**。它可以用极其自信、极其专业的语气给你一段完全错误的信息。你如果不去验证，根本看不出来有问题。

Claude至少在不确定的时候会明确告诉你"我不确定"。这种诚实在工作中太重要了——给我一个不准确但标注了"不确定"的信息，远比给我一个看似准确但其实是编的信息好得多。

那时候的感受：**可靠性才是第一位的！**

## 目前的用法：双修才是正道

折腾了快两年我终于想明白了——**不要选一个，两个都用**。两个工具的优势领域完全不同，互补而不是替代。

| 场景 | 用谁 | 具体原因 |
|------|------|----------|
| 写代码（复杂项目） | Claude | 长上下文稳，不瞎编API，代码风格一致 |
| 写代码（快速原型） | ChatGPT | 速度快，不啰嗦，Canvas里直接改 |
| 搜索最新信息 | ChatGPT | 能联网，有浏览功能 |
| 生成图片 | ChatGPT | DALL·E 集成 |
| 写文章/长文档 | Claude | 文风更自然，结构更好，不会写着写着突然变风格 |
| 数据分析 | ChatGPT | Code Interpreter 真的好用 |
| Code Review | Claude | 能准确理解大量上下文，指出逻辑问题 |
| 头脑风暴 | 都行 | 各有千秋，有时候两个的答案互相补充效果更好 |
| 学习新技术 | ChatGPT | 联网能查最新资料 |
| 重要决策参考 | Claude | 更谨慎，不容易被幻觉误导 |

两个工具加起来每个月花费 40 刀（ChatGPT Plus $20 + Claude Pro $20），说多不多说少不少。但提升的效率确实值这个价——保守估计每个月帮我省了至少30-40小时。

### 我的具体工作流

1. **日常编程**：Claude作为主力。在VS Code里用Continue插件连接Claude API，每天的代码辅助、code review、debug都用它
2. **搜索和研究**：ChatGPT。需要了解某个新技术、查某个错误信息的最新解决方案时用
3. **写文档和博客**：先用Claude写初稿（它的文风更自然），然后用ChatGPT的Canvas编辑修改
4. **数据分析**：ChatGPT的Code Interpreter，直接上传CSV/Excel，让它跑Python分析
5. **做图**：ChatGPT + DALL·E，快速生成概念图、配图

## 它们在2025-2026的进化

值得一提的是，两个工具在这段时间都有很大进步：

### ChatGPT的变化
- GPT-5系列比GPT-4o聪明了不少，但幻觉问题依然没有根本解决
- Atlas浏览器让搜索体验更无缝
- Canvas编辑器持续改进
- 加入了Agent能力，可以执行多步骤任务
- 但开始加广告了，体验有点下降……

### Claude的变化
- Claude 3.5系列在编程和推理上真的很能打
- 加了网络搜索功能（虽然还不完美）
- Artifacts功能越来越好用
- Project Knowledge让长期项目的上下文管理更方便
- 依然没有图片生成能力

## 给还在纠结的人的建议

### 1. 不要有信仰

工具就是工具，哪个好用用哪个。我见过ChatGPT的"死忠粉"各种为它的幻觉辩护，也见过Claude的信徒宁可手动搜索也不用ChatGPT的联网功能。这都没必要。

### 2. 先用免费版感受

ChatGPT和Claude都有免费版。ChatGPT免费版现在用的是较弱的模型，体验跟Plus差不少，但足以感受基本的交互风格。Claude的免费版额度有限但模型跟付费版一样。

建议用同样的问题分别问两个，自己感受一下差异。特别是编程类问题和创意写作类问题，两者的差异很明显。

### 3. 根据你的核心场景选择

| 你的主要需求 | 推荐 |
|-------------|------|
| 编程为主 | Claude（如果预算只够一个） |
| 写作为主 | Claude |
| 搜索/信息为主 | ChatGPT |
| 多媒体（图片/图表） | ChatGPT |
| 需要最强综合能力 | 两个都订 |

### 4. 两个都订没毛病

一个月多花 20 刀（大概140块人民币），省的时间远不止这个价。如果你的工作涉及到多种AI使用场景（大部分知识工作者都是），双修是最优解。

### 5. 不要忽视国产选项

如果你主要在中文环境下使用，豆包（字节跳动）、Kimi（月之暗面）、通义千问（阿里）这些国产AI也值得一试。中文理解能力有些场景不输ChatGPT和Claude，而且很多功能免费。特别是Kimi的长文本能力和通义的中文写作，水平已经很不错了。

## 一个有意思的发现

用了两个AI之后，我发现了一个有趣的现象：**当我把两个AI的回答互相喂给对方时，最终的结果往往比单独用任何一个都好**。

比如：
1. 让ChatGPT搜索某个技术方案的最新实践
2. 把ChatGPT找到的信息喂给Claude，让Claude深入分析
3. Claude给出分析后，再把分析结果给ChatGPT，让它用Code Interpreter实际验证

这个A→B→A的流程虽然多花了点时间，但在重要决策上效果非常好。两个AI互相"质疑"和"补充"，有点像pair programming但更快。

## 最后补一句

那些说自己只用某一家的人，大概率是没认真用过另一家。AI工具跟编程语言不一样——编程语言有生态锁定效应，但AI工具的切换成本几乎为零。多试试没坏处。
