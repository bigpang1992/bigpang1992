我在 ChatGPT 和 Claude 之间来回切换了至少四次。每次切换都觉得"这次终于找到真爱了"，然后过两周又开始觉得"还是那个好"。

今天来复盘一下这段堪比渣男的经历。

## 第一阶段：ChatGPT 初恋

2024 年，跟大部分人一样，ChatGPT 是我的第一个 AI 工具。当时觉得太牛了，什么都能答，什么都能写，感觉整个世界都被打开了。

那时候的感受：**这就是未来啊！**

## 第二阶段：转投 Claude

后来开始写代码用 AI，发现 ChatGPT 在长代码场景下有个毛病——越聊越傻。上下文长了之后，前面说好的约定它全忘了，代码风格也开始飘。

朋友推荐我试 Claude，我一试就惊了。Claude 对长上下文的理解能力明显更强，写代码也更稳重。最关键的是——它会拒绝你。不确定的东西它会说"我不确定"，而不是像 GPT 那样自信满满地胡扯。

那时候的感受：**之前浪费的时间都是什么！**

## 第三阶段：又回 ChatGPT

用了一段时间 Claude 之后，发现它也有明显短板：

- 不能联网搜索（那时候还不行）
- 不能生成图片
- 插件生态远不如 GPT
- 有时候太啰嗦，废话一大堆

于是又回了 GPT。特别是 GPT 加了Canvas功能之后，编辑文档的体验确实很好。

那时候的感受：**果然还是 ChatGPT 全面！**

## 第四阶段：再回 Claude

GPT 用了一阵子，历史毛病又犯了——幻觉。

有一次我让它帮我查一个 npm 包的用法，它信誓旦旦地给了我一段示例代码。我照着抄，怎么都报错。查了半天发现——那个 API 根本不存在，是它编的。

同样的问题问 Claude，它诚实地说："这个包我不太确定最新版本的API变化，建议查看官方文档。"

就这一个对比，我又回去了。

那时候的感受：**可靠性才是第一位的！**

## 目前的用法

折腾了这么久我终于想明白了——不要选一个，两个都用。

| 场景 | 用谁 | 原因 |
|------|------|------|
| 写代码 | Claude | 长上下文稳，不瞎编 API |
| 搜索信息 | ChatGPT | 能联网，有浏览功能 |
| 生成图片 | ChatGPT | DALL·E 集成 |
| 写文章 | Claude | 文风更自然，结构更好 |
| 分析数据 | ChatGPT | Code Interpreter 好用 |
| 头脑风暴 | 都行 | 各有千秋 |

两个工具加起来每个月花费 40 刀，说多不多说少不少。但提升的效率确实值这个价。

## 给还在纠结的人的建议

1. **不要有信仰** — 工具就是工具，哪个好用用哪个
2. **先用免费版感受** — 都有免费额度，试了再说
3. **关注你的核心场景** — 写代码和写文案对AI的需求完全不同
4. **两个都订没毛病** — 一个月多花 20 刀，省的时间远不止这个价

最后补一句：那些说自己只用某一家的人，大概率是没认真用过另一家。
