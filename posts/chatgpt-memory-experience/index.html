ChatGPT的记忆功能用了两周，我有被冒犯到

📅 2026年03月12日 ⏱ 1 分钟阅读 🏷️ AI, ChatGPT, 隐私, 体验
📢 推广
📢

广告位虚位以待，联系站长投放

ChatGPT 的记忆功能上线挺久了，但我一直没怎么在意。直到最近，它做了一件让我浑身发毛的事。

这篇文章记录一下我跟ChatGPT的记忆功能"斗智斗勇"的两周，有惊喜也有惊吓。

事情是这样的

某天我随口跟 ChatGPT 吐槽：“今天又吃泡面了，穷得叮当响。”

就是随口一说，那天就是不想点外卖而已。

过了大概一周，我问它帮我推荐个午餐菜谱。它回复：

“考虑到你最近经济压力比较大，给你推荐几道成本在 5 块钱以内的菜谱……”

我当时就愣住了。

什么叫"考虑到我最近经济压力比较大"？？？我只是吃了一碗泡面啊！你是不是对我有偏见？？？

这就好比你跟朋友随口说了一句"今天没吃早饭"，一周后他送你一袋大米说"你最近是不是经济困难"。

翻了一下记忆列表：好家伙

好奇心驱使下，我去翻了它的记忆列表（设置 → 个性化 → 记忆）。好家伙，密密麻麻记了一大堆：

合理的部分：

  • “用户是程序员，主要使用 JavaScript”（这个正常，聊代码的时候说过很多次）
  • “用户使用 React 18 + TypeScript，项目用 Vite 构建”（对，省得每次都重复了）
  • “用户偏好简洁的代码风格，不喜欢过度封装”（这个甚至有点贴心）

有点私密的部分：

  • “用户喜欢在深夜提问”（确实，但你不用特意记吧……）
  • “用户养了一只猫叫『大橘』"（有一次提了一嘴而已）
  • “用户对煮粥有执念”（我就问了两次粥的做法！两次！）

让我不适的部分：

  • “用户近期经济压力较大”（泡面事件的后遗症）
  • “用户对工作现状有些不满”（某次加班后的牢骚被它记住了）
  • “用户周末习惯宅在家里”（这是在说我没有社交生活吗）

看完之后我的心情非常复杂。这些记忆单独拿出来都没什么，但合在一起，它比我妈都了解我。而且它形成了一个"人设”——在AI眼里，我是一个深夜加班、经济拮据、喜欢煮粥的孤独宅男程序员（虽然好像也没说错……）。

记忆功能的底层逻辑

研究了一下，ChatGPT的记忆功能运作方式大致是：

  1. 自动提取：它在对话中自动识别值得记忆的信息（你不需要主动说"记住这个"）
  2. 结构化存储：每条记忆都是一个简短的文本描述
  3. 主动调用：后续对话中，它会自动关联相关的记忆来优化回答
  4. 用户可管理：你可以查看、编辑、删除单条记忆

问题在于第一步——它自动决定记住什么。你无法提前设置"哪些话题不要记"，它就像一个特别爱记笔记的朋友，你的每句话它都可能悄悄记下来。

而且它的"记忆理解"能力有时候很离谱。我吐槽一句泡面，它就跳跃到"经济压力"。我抱怨一次加班，它就总结为"对工作不满"。这种过度解读的记忆，会反过来影响它后续回答的方向。

方便是真的方便

吐槽归吐槽，记忆功能确实好用，甚至可以说是ChatGPT最贴心的功能之一。

不用重复上下文了

以前跟它聊代码，每次都得说：“我用的是 React 18 + TypeScript + Vite，项目结构是这样的……” 重复了不下五十次，烦得要死。现在它直接就知道了，每次上来就能接着上次的语境聊。

代码风格匹配了

它记住了我喜欢简洁的代码风格，不喜欢过度封装。所以现在它生成的代码明显比以前"合我胃口"了。以前它喜欢给你来一套factory pattern + abstract class + interface的全家桶。现在知道我就喜欢一个简单的函数搞定。

个性化推荐更准了

问它推荐技术书籍，它会根据我的技术栈推荐具体方向的书。问它推荐做饭方法（虽然我只是偶尔做），它知道我家里没有烤箱只有微波炉。

对话的连续性增强了

上周我聊到一半关了页面。第二天创建新对话继续聊同一个项目bug，它居然能回忆起之前讨论过的上下文。这在以前是不可能的——每个新对话都是"失忆状态"。

但隐私焦虑也是真的

这东西本质上就是一个会记住你所有隐私的聊天对象。而且跟人类朋友不同，它的记忆是系统化的、永久的、可被公司访问的。

几个让我不安的点

1. 数据存在哪里？

记忆数据存在 OpenAI 的服务器上。虽然OpenAI声称有数据保护措施，但这些信息本质上在第三方手中。如果服务器被攻击、数据泄露，你所有的"画像"都会暴露。

2. 是否用于训练？

OpenAI说可以选择关闭数据训练，但记忆功能本身的工作机制和训练数据之间的界限不够清晰。你的记忆条目是否会以某种匿名化的形式进入训练集？这个问题没有确切答案。

3. 无法完全掌控

你可以删除单条记忆，但你无法知道AI在"理解"层面记住了多少。也许你删了那条"经济压力"的记忆，但它在模型内部的权重调整可能还在。这种不透明性是最让人不安的。

我后来养成的保护习惯

  1. 聊完敏感话题就去记忆列表里删一下
  2. 涉及到真实姓名、地址、财务信息的话题用临时聊天
  3. 定期（大约两周一次）去清理明显不该记住的条目
  4. 跟ChatGPT聊天时有意识地控制信息暴露

但这本身就说明了问题——你在用一个你不完全信任的东西，还得花时间管理它对你的了解程度。这种关系本身就挺诡异的。

最离谱的几次经历

“你不是一般都宅在家吗”

上周我问它：“帮我想个周末计划。”

它说：“你周末不是一般都宅在家里吗？还是安排点室内活动吧。比如试试新的粥的做法？你之前不是对煮粥很感兴趣吗。”

我……谢谢你的坦诚和关怀，但我没让你拆穿我的社交生活。

猜我的心情

有一次下午三点多我突然问了一个很基础的CSS问题（居中对齐）。它回复的时候开头加了一句：“你今天是不是有点累？这个问题你平时应该不需要问的。”

虽然确实累了（那天连续开了四个会），但被AI看穿这件事让我有种被监视的不适感。它把"你平时问的问题难度"也当成了了解你状态的线索——这个观察力真的有点过头了。

记忆冲突

有一次它的两条记忆打架了。一条记着"用户喜欢吃辣"（我某次提过想吃火锅），另一条记着"用户胃不太好"（某次随口说的）。结果它推荐菜谱的时候非常纠结——给了一个"微辣但不伤胃"的折中方案。

你别说，这个处理方式还挺人性化的。

和其他AI的记忆功能对比

ChatGPT不是唯一有记忆功能的AI。Claude最近也推出了类似功能，Google Gemini也有。简单对比一下：

功能ChatGPTClaudeGemini
自动记忆✅有限✅
手动管理✅ 方便✅⚠️ 不够直观
过度解读程度高低中
跨对话记忆✅有限✅

Claude的记忆相对克制很多，不会像ChatGPT那样大量自动推断。但也因此不够"聪明"。Gemini配合Google生态的数据，理论上能记住更多，但管理界面不够友好。

使用建议（认真的）

如果你也在用ChatGPT的记忆功能，或者准备开启：

  1. 定期检查记忆列表 — 设置里可以查看所有记忆条目，至少每两周清理一次
  2. 敏感话题用临时聊天 — 不想被记住的内容用临时会话，或者在主题聊完后主动说"请忘记这次讨论的内容"
  3. 不要完全依赖它的判断 — 它对你的"画像"未必准确，可能因为一句随口的话就给你打上某个标签
  4. 利用记忆功能提效 — 与其被动让它记，不如主动告诉它你想让它记住什么（比如"记住：我的项目用的是Next.js 14"）
  5. 了解删除的局限性 — 删除记忆条目不代表完全"忘记"，模型可能在其他层面保留了信息

写在最后

记忆功能让ChatGPT从"工具"更接近了"助手"。但也让它离"监控"更近了一步。

这两者之间的距离，取决于你怎么使用它，以及你对OpenAI有多少信任。

我的选择是：开着记忆功能（因为确实方便），但保持警觉。就像你会跟一个聪明但你不完全信任的同事合作——保持合理的信息暴露，不说不该说的话。

而且再也不在ChatGPT面前吃泡面了。

你用记忆功能有什么有趣的经历吗？有没有被AI看穿过什么？来评论区分享一下你的故事。

记忆功能是个双刃剑。用好了是贴心助手，用不好就是电子监护人。各位自己拿捏吧。

写完这篇文章我去检查了一下记忆列表，发现它新增了一条：“用户正在写关于ChatGPT记忆功能的文章”。

……行吧。
